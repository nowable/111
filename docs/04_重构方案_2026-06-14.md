## 0. 一句话结论
**不是从零重构——lane-follow + 事件状态机已存在。** 昨天兜圈是因为 `route_mode=map2d` 走了 odom 航点路径(`_execute_map_route`)。本方案 = **关掉 map 路径让任务落到现成 vision/lane 分支 + 补 3 个洞 + 打通真 LLM + 调参**。

## 1. 现状（已勘探确认）
- 代码：本地 10 .py == 板上 src；改 `src/.py` 即生效(symlink-install)+ **必须重启进程**；`build/lib` 旧拷贝是陷阱、与运行无关。
- 状态机(已有)：IDLE→PREFLIGHT→DRIVE_TO_QR→SCAN_QR→DECIDE_DIRECTION→CORRIDOR_FOLLOW→LOOP_L/R→CAPTURE_TARGET_IMAGE→CALL_LLM_API→RETURN_HOME→STOP。
- 每个驾驶状态有 3 分支：map2d / fixed2d / **else=纯 vision-lane**(当前因 route_mode=map2d 未被走到)。
- `LaneFollower`(lane_follow.py)：黄掩膜 ROI 质心 PID，center/left_edge/right_edge，已可用。
- `_state_drive_to_qr` else 分支 = 奔二维码板 + 躲桶(已存在)。

## 2. 目标架构
**全局避障层(YOLO+dark_blue，全程常开)叠加在 每段转向意图 上：**
- 大厅：奔二维码(已有)→扫码→**奔通道口(新桥)**，全程躲桶。
- 通道/环：LaneFollower 黄道(已有)+ **加避障(新)**。
- C 环上侧：路过抓牌 → qwen-vl 描述(真调，非 placeholder)。
- 返航：**lane 反向下通道 + 视觉认 P 停车(新，去 odom 依赖)**。

## 3. 改动分块（Codex 任务，串行；每块我审 diff 后部署）

### Chunk 1 — 关 map 路径 + 大厅 QR→通道口桥 + 端到端 lane 流
- config(板上 surgical)：`route_mode` 改非 map2d/fixed2d(如 `"lane"`)→ 所有 `_map_route_ready/_fixed_route_ready` 返 False，落 else。先确认 `MissionConfig.validate()` 不拒该值；拒则改 `map_route_enabled=false`+`fixed_route_enabled=false`。
- 代码：DRIVE_TO_QR 扫码确认方向后，新增"奔黄色通道口+躲桶"衔接(朝最大黄连通域质心转向，黄占比够/见绿→切 CORRIDOR_FOLLOW)。
- 验收：dry-run 状态流走到 DONE、exit 0。

### Chunk 2 — 全局避障层（叠加到 lane 循环）
- 抽 `_avoid_step(image, decision) -> Optional[(lin,ang)]`：danger 时返回"让向空侧"，否则 None。
- 织入 `_state_corridor_follow` 与 `_run_ring_follow`：每帧先 _analyze_frame，danger→发避让 continue，否则 LaneFollower。**让向永远偏离绿色一侧**(环内侧是绿→只能往外让)。
- config：`obstacle_avoid_enabled=true`。
- 验收：板上喂含桶图触发避让、且不向绿。

### Chunk 3 — 环完成判定 + 返航 + P 识别停车（去 odom 依赖）
- 环完成：`_run_ring_follow` 不只靠 integrated_yaw(会瞬跳)；加"最小时长+抓到牌+回到起点附近黄"复合判据。
- 返航：lane 跟黄反向下通道；到大厅 **视觉认 P(深蓝方块+白色边框，区别于无白框的蓝桶)** 朝其行进、占比够近即停。
- 验收：板上喂 P 图能检出、喂桶图不误判。

### Chunk 4 — 打通真 LLM 读牌（完赛硬指标）
- config：`llm_mode` placeholder→`openai-compatible`；确认板上 `DASHSCOPE_API_KEY` 已设、热点联网可达。
- 加超时重试；两块同牌=两次机会兜底；读不到不阻塞其余项但记录。
- 验收：板上真发一张牌图，qwen-vl 返回中文描述。
