## 结论（驱动了/挡住了哪些改动）

1. **green_floor 误退 = 单帧尖峰**：所有真实帧 green ≤ 0.0023，**稳态远不及 0.15 阈值**。
   故唯一能触发 green_floor 的只能是瞬时坏帧（反光/模糊/旧 bridge bug 把车留在
   非通道场景）。→ 已加 `green_exit_frames=3` 连续帧去抖（commit 3fb3258），
   证据支持：稳态永不触发，去抖只挡瞬时尖峰，不会误伤真正到环的整片绿。

2. **拍牌 marker 误触发：离线无法复现**。orange 目标牌在**所有**帧（含带橙色发布台垫的
   scene_qr）都是 **0 个** ≥0.025 的检出。说明现 orange HSV + 0.025 面积 + 0.35-3.2 长宽比
   在手头帧上不误报。“环里拍 9 次假牌”极可能是**旧 bridge bug 顶死立牌后状态机空跑
   LOOP_RIGHT 的产物**（污染数据），不是真·环内误检。
   → **事实驱动：在拿到真实 C 环帧之前，绝不动 marker 阈值**（盲调会赔上 读牌 硬指标）。
   下次场地务必存几张**真·环上侧正对图文牌**的帧，再回来判。
   注：`detect_target_cards` 用 **全帧**(roi_y_ratio=None)；真牌在环上侧**顶角**、橙字在
   地面下部——若现场确认地面橙字误检，**几何修法**=只认画面上半区的橙块（待证再做）。

3. **bridge 通道口 yellow 门槛 0.06 位置正确**：mouth.jpg(顶死立牌)=0.038 < 0.06 <
   corridor_first(真通道接近)=0.107。假场景在门下、真通道在门上 → 0.06 不动。
   叠加新 `line_acquired` 闩锁，橙垫单独再也假报不了口。

4. **black 线**：scene_qr=0.055、b1=0.026、corridor_first=0.024、qrfail=0.022 在
   lane_min_mask_area_ratio=0.01 之上 → 黑线循迹 lane_found 成立，符合预期。

5. **lane_mask_erode_px 仍无法离线定标**：手头无“1m 实心黄道 + 两侧黄白网格墙”同框帧，
   corridor_first/last 只有实心黄、无网格。→ 维持默认 0（关）；下次现场存一张
   **车在 1m gap 内、两侧网格入镜**的帧再定 erode 核大小。
