# OriginCar 当前架构说明 2026-06-10

这份文档描述当前代码实际状态，重点覆盖 6 月 10 日新增的固定地图路线、shunt 原地近似转向、二维码后水平向左对齐、外场数据采集流程。旧版索引仍保留在 `ARCHITECTURE.md`，但其中部分流程已经过期。

## 1. 项目目标

车从 P 点出发，沿 A 区黑色虚线接近二维码，扫码决定 C 区绕圈方向；扫码后先把车头对齐到地图水平向左，再进入 B 区黄色通道，进入 C 区绕环，拍当前方向对应的图文牌，调用 LLM 识别，最后按地图返航路线回到 P 点。

当前主流程：

```text
P_START
  -> A_TO_QR
  -> SCAN_QR
  -> DECIDE_DIRECTION
  -> QR_ALIGN_HEADING(180deg, shunt)
  -> QR_TO_CORRIDOR
  -> B_CORRIDOR
  -> C_LOOP_LEFT 或 C_LOOP_RIGHT
  -> CAPTURE_TARGET_IMAGE
  -> CALL_LLM_API
  -> RETURN_TO_P
  -> STOP/DONE
```

## 3. 状态机架构

主状态机在 `auto_mission.py`：

```text
STATE_SEQUENCE
AutoMissionNode
MissionConfig
```

状态顺序：

```text
IDLE
PREFLIGHT
DRIVE_TO_QR
SCAN_QR
DECIDE_DIRECTION
CORRIDOR_FOLLOW
LOOP_LEFT / LOOP_RIGHT
CAPTURE_TARGET_IMAGE
CALL_LLM_API
RETURN_HOME
STOP
DONE
```

每个状态对应一个 `_state_*` 方法。

关键状态说明：

| 状态 | 当前行为 |
|---|---|
| `DRIVE_TO_QR` | 优先执行 `map2d` 的 `A_TO_QR` 航点路线，同时检测二维码和障碍 |
| `SCAN_QR` | 停车扫码，连续确认方向，锁定后不再跳变 |
| `DECIDE_DIRECTION` | 解析/确认 left/right，然后执行 `QR_ALIGN_HEADING`，再走 `QR_TO_CORRIDOR` |
| `CORRIDOR_FOLLOW` | 优先执行 `B_CORRIDOR` 地图路线，可叠加黄线纠偏 |
| `LOOP_LEFT/RIGHT` | 根据二维码方向选择 `C_LOOP_LEFT` 或 `C_LOOP_RIGHT`，只捕获对应图文牌 |
| `RETURN_HOME` | 优先 `RETURN_TO_P` 地图返航，失败再 trajectory fallback |

## 4. 内置 2D 地图

地图模块在 `route_map.py`：

```text
RouteMap
Pose2D
Waypoint
DEFAULT_ROUTE_MAP
map_pose_from_odom
```

坐标约定：

```text
地图尺寸：5m x 5m
原点：A 区左下角
x：向右
y：向上，朝 B/C 区
```

当前路线：

| 路线名 | 用途 |
---|---|
| `A_TO_QR` | P 点到二维码，贴用户红线/A 区黑虚线 |
| `QR_TO_CORRIDOR` | 二维码位置回到中间通道入口 |
| `B_CORRIDOR` | B 区通道 |
| `C_LOOP_RIGHT` | C 区右环 |
| `C_LOOP_LEFT` | C 区左环 |
| `RETURN_TO_P` | 第二条黑虚线返航到 P |

路线执行在 `auto_mission.py`：

```text
_execute_map_route
_current_map_pose
_reset_map_frame
_map_route_timeout
```

`map_pose_from_odom()` 用启动时 odom 位姿和配置里的 `map_start_pose` 建立地图坐标。起点摆放角度非常重要。

## 5. 二维码后水平向左对齐

这是 2026-06-10 新增的关键流程。

配置键：

```json
{
  "qr_align_after_scan_enabled": true,
  "qr_exit_heading_deg": 180.0,
  "qr_align_heading_tol": 0.10
}
```

实现位置：

```text
auto_mission.py
  _state_decide_direction
  _align_after_qr_scan
  _rotate_relative_arc
```

执行逻辑：

1. `SCAN_QR` 解码并锁定方向。
2. 进入 `DECIDE_DIRECTION`。
3. 如果 `qr_align_after_scan_enabled=true`，读取当前 map pose。
4. 计算当前 yaw 到 `qr_exit_heading_deg=180` 的相对角度。
5. 默认用 shunt 原地近似转向。
6. 对齐成功后执行 `QR_TO_CORRIDOR`。

dry-run 应出现：

```text
STATE: DECIDE_DIRECTION
ACTION: QR_ALIGN_HEADING target=180.0deg mode=shunt tol=0.10rad
MAP_ROUTE_START name=QR_TO_CORRIDOR
```

真车日志应出现：

```text
QR_ALIGN_HEADING_START target_deg=180.0 current_deg=... delta_deg=... mode=shunt
QR_ALIGN_HEADING_DONE reason=yaw turned_deg=... elapsed=... cycles=...
```

## 6. Shunt 原地近似转向

前轮转向/后轮推进底盘不能差速原地旋转，因此新增 `ackermann_turn.py`：

```text
execute_shunt_turn(...)
ShuntTurnResult
estimated_turn_time_s(...)
```

函数接口核心参数：

```python
execute_shunt_turn(
    target_angle_rad,
    motion_state=...,
    publish_velocity=...,
    spin_sleep=...,
    forward_linear=...,
    reverse_linear=...,
    angular=...,
    heading_tol=...,
    timeout=...,
    pulse_s=...,
    stop_s=...,
)
```

控制思想：

```text
目标左转：前进 + 正 angular.z，停，后退 + 正 angular.z，停，重复
目标右转：前进 + 负 angular.z，停，后退 + 负 angular.z，停，重复
```

注意：`Twist.angular.z` 表示期望车身 yaw 方向，不是方向盘角度。因此前进和后退使用同号 `angular.z`。

当前默认速度配置：

```json
{
  "ackermann_turn_mode": "shunt",
  "shunt_forward_linear": 0.06,
  "shunt_reverse_linear": 0.06,
  "shunt_pulse_s": 1.0,
  "shunt_stop_s": 0.2,
  "shunt_turn_timeout": 45.0,
  "fixed_route_turn_angular": 0.4,
  "return_turn_angular": 0.4
}
```

调用点：

| 调用点 | 用途 |
---|---|
| `square_drive_test.py` | 测固定轨迹和 shunt 转向能力 |
| `_fixed_route_turn` | fixed2d 定角转 |
| `_rotate_to_heading` | trajectory 返航对齐航向 |
| `_align_after_qr_scan` | QR 后水平向左对齐 |

安全保护：

- `execute_shunt_turn()` 内部有 `finally`，退出时发布 stop。
- `square_drive_test` 捕获 SIGINT/SIGTERM，Ctrl+C 后连续发 stop。
- 如果底盘仍保持最后速度，手动发布 0 速度：

```bash
timeout 3 ros2 topic pub -r 20 /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
```

```bash

  --output-dir /root/dev_ws/datasets/origin_competition \
```

## 10. 配置文件职责

主要配置：

```text
config/mission_defaults.json
```

带注释版本：

```text
config/mission_defaults.annotated.jsonc
```

原则：

```text
外场只改 mission_defaults.json
代码里默认值只作为兜底
运行 auto_mission 必须带 --config $CFG
```

外场最常改：

| 目标 | 配置键 |
---|---|
| shunt 转快/慢 | `shunt_forward_linear`, `shunt_reverse_linear`, `fixed_route_turn_angular`, `return_turn_angular` |
| QR 后对齐开关 | `qr_align_after_scan_enabled` |
| QR 后目标朝向 | `qr_exit_heading_deg` |
| map2d 速度 | `map_max_linear`, `map_max_angular` |
| 航点到位容差 | `map_waypoint_tol` |
| 黄线 HSV | `color_profiles.yellow` |
| 深蓝障碍 HSV | `color_profiles.dark_blue` |
| 橙色牌 HSV | `color_profiles.orange` |
| 绿色禁区 HSV | `color_profiles.green` |

## 11. 当前已知风险

1. shunt 转向依赖底盘接受负 `linear.x`。如果倒车转向不明显，需要回退到 `ackermann_turn_mode=arc`。
2. map2d 依赖 P 点摆放方向。起点角度不准，后面路线都会偏。
3. 颜色 HSV 必须在外场重新确认。宿舍光照下的阈值不能直接信。
4. YOLO 目前不是唯一避障来源。障碍桶数据采集后还要标注和训练。
5. Ctrl+C 后底盘可能保持最后速度，必须准备急停命令。
