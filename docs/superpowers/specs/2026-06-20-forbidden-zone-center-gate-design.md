# 禁区中心候选带设计

## 目标

解决扫码后桥接到黄色入口这段，车辆会把两侧黄白相间禁区误当成可走黄色区域并直接撞上去的问题。

最新现场结论是：不再依赖“区分禁区和黄色入口块”的视觉逻辑，而改成最简二值标准：

- 黄色 = 已经到禁区边
- 黑线 = 该右转进入黄色区域

进一步的现场流程要求是：

- 倒车后固定前推 3 秒
- 黄色出现后，不是一直左转沿禁区跑，而是先左转到与禁区平行
- 左转到位后再切成直走找黑线
- 黑线一旦出现，立刻以最大角速度右转 90 度入区，不允许再停顿

最终“与禁区平行”的判定不再靠主观感觉，而是以参考图中黄色禁区主块的位置作为数值标准。

当前参考图计算结果：

- 黄色主禁区块 `offset_norm ≈ 0.545`

因此本次平行判定目标取：

- `bridge_yellow_parallel_target_offset = 0.55`
- `bridge_yellow_parallel_tol = 0.10`

当黄色主块偏移进入 `0.45 ~ 0.65` 时，认为车身已经基本与禁区平行，可以切成直走找黑线；超出这个范围则继续回正。

后续现场又明确两点动作要求：

1. 倒车后的前推阶段不能纯直走
   - 需要带一个小左转
   - 方便更快对齐禁区边界方向

2. 黑线触发后的入区动作不能原地空转
   - 需要带前进速度形成右转弧线
   - 目标转角改为 `120` 度
   - 风格偏激进，保证“能真正拐进去”

后续流程上，用户明确要求删除绕圈后的抓图识别与 LLM 调用，因此环绕完成后不再走 `CAPTURE_TARGET_IMAGE -> CALL_LLM_API`，而是直接切入 `RETURN_HOME`。

返航主线也不再依赖轨迹回放，而改成黑线循迹返航：

- 进入 `RETURN_HOME` 后直接创建黑线 `LaneFollower`
- 沿着黑线持续前进返航
- 一旦黑线丢失，直接停车结束

也就是说，返航的结束标准不再是“回放完整轨迹”，而是“黑线丢失即停”。

## 当前问题

当前 `_find_bridge_corridor_target()` 的逻辑是：

1. 提取黄色 mask
2. 对黄色做腐蚀，尽量压掉禁区的细黄条
3. 对剩余黄色连通块按面积、靠下程度、居中程度打分
4. 选分数最高的黄色块作为入口目标

问题在于这仍然是“在所有黄色块里猜哪个像入口”，而不是“先排除禁区，再选入口”。

一旦侧边禁区残留的黄色块仍然：

- 面积够大
- 靠画面下方
- 偏移没有大到被严重惩罚

它就仍然可能被选中，导致车辆朝禁区撞过去。

## 方案

桥接主逻辑改为“黄区判禁区、黑线触发转入黄区”。

新的桥接流程：

1. 扫码后先倒车脱离二维码牌
2. 固定前推 3 秒
3. 如果还没看到黄色，边走边右转搜索禁区
4. 一旦看到黄色，认为已经到禁区边，先左转到与禁区平行
5. 左转到位后改成直走找黑线
6. 一旦重新看到黑线，立刻触发一次最大角速度右转
7. 右转完成后直接进入黄色安全区流程

也就是说，桥接阶段不再需要判断“哪个黄色块才是入口”，而只需要：

- 看到黄色：说明禁区到了
- 看到黑线：说明入口到位，可以右转入区

## 改动范围

只改 `QR -> B区` 桥接阶段逻辑和配置项：

- 在桥接阶段用黄色面积判断“禁区是否出现”
- 在桥接阶段复用前面大厅用过的黑线识别能力
- 增加黑线触发右转入区动作
- 保留现有：
  - `BRIDGE_BACKUP`
  - `BRIDGE_POST_BACKUP_PULSE`
  - 超时与基础搜索阶段

不改：

- `CORRIDOR_FOLLOW`
- 其它 YOLO / 避障主线

## 新配置

新增或改造的桥接阶段参数会围绕三阶段展开，重点包括：

- 固定前推时长/速度
- 搜黄阶段右转速度与角速度
- 黄区触发阈值
- 黑线搜索 ROI
- 黑线触发最小面积/连续帧
- 右转角度与角速度
- 右转完成后的入区推进量

## 数据流

桥接阶段流程变成：

1. 取最新图像
2. 固定前推 3 秒
3. 若黄色未出现，边走边右转搜索禁区
4. 若黄色出现，先做一次固定左转对齐禁区
5. 左转到位后复用黑线 `LaneFollower(..., 'black', ...)` 或其 mask 能力判断黑线是否重新出现
6. 若黑线出现并满足触发条件，立刻执行最大角速度右转约 90 度
7. 右转完成后直接切入黄色区域流程

## 失败保护

这次修正的核心安全目标是：

- 黄色出现前，不要盲目直冲
- 黄色出现后，不要无限左转，必须在左转到位后切成直走
- 黑线出现前，不要提前右转入区

因此行为优先级改为：

1. 不撞禁区
2. 用黄色确认“禁区已到”
3. 左转到与禁区平行
4. 再直走稳定接近入口黑线
4. 用黑线确认“该右转了”
5. 再统一右转进入黄色区

## 测试与验证

验证重点改成：

1. 禁区附近不再依赖黄色入口块判断
2. 黄色没出现时，车辆边走边右转搜索
3. 黄色出现后，车辆先左转对齐，再直走找黑线
4. 黑线一旦出现，车辆立刻执行统一右转进入黄色区
5. 右转后能顺利衔接黄色区域程序

后续现场反馈说明当前这版仍有两个问题：

1. 黄色禁区触发过迟，车辆会直接闯过去
2. 直走找黑线阶段不够灵敏，且速度偏慢

因此本轮最小调优目标是：

- 让黄色更早触发
- 让黑线更容易确认
- 让直走找黑线阶段更快

本轮只调这三类量：

- `bridge_yellow_detect_min_ratio` 调小
- `bridge_black_reentry_confirm_frames` 调低
- `bridge_black_search_linear` 调高

现场日志应重点观察：

- `BRIDGE_BACKUP`
- `BRIDGE_POST_BACKUP_PULSE`
- 黑线重新出现的触发日志
- 右转入区触发日志

## 取舍

本次不再继续堆“黄区/禁区区分”逻辑，也不做更复杂的黄白纹理检测。

原因：

- 现场已经确认：黄色只用来判断禁区已到，入口位置最终由黑线更可靠地指示
- 继续加黄色块/禁区几何规则会越来越复杂
- 用黑线重新接管桥接，更接近真实赛道结构，也更利于现场调参
