## 当前结论

1. `use_imu_yaw=false` 后，`A_TO_QR` 起步到 `A_DASH_04` 基本能贴合地图路线。
2. 最新失败不是 Plan A 叠加坏了，而是自动控制在 `A_DASH_04 -> A_DASH_05` 段后过度转向，位姿从 `x=2.262 y=0.475 yaw=7.7` 跳到 `x=2.627 y=1.100 yaw=108.9`，随后对 `A_DASH_05` 的 `heading_err` 接近反向，控制器以 `angular=-0.250` 饱和右打，表现为偏离后转圈。
3. 现场下一步优先调低 map2d 航点跟踪的转向增益和最大角速度，而不是继续改路线点。

## 已做/已建议的现场改动

### 1. 关闭起步航向保护

问题现象：

```text
MAP_START_HEADING_MISMATCH route=A_TO_QR waypoint=A_DASH_01
failure_reason: map start heading mismatch
```

原因：

- `map_start_heading_guard` 原本用于防止车头放反后撞墙。
- 当前实现只要目标仍是 `A_DASH_01` 就持续检查，运行十几秒后也可能误停。

现场处理：

```json
"map_start_heading_guard_enabled": false,
"map_start_heading_guard_rad": 9.9
```

注意：本地 `origin_competition_auto/config/mission_defaults.json` 当前仍可能是默认开启状态，需回收板子配置后同步。

### 2. 手控网页左右键从原地转改为弧线转

问题现象：

- 9898 手控网页前进/后退有反应。
- 左右键无反应。

原因：

- 原网页左右键发的是 `linear_x=0, angular_z=±omega`。
- OriginCar/阿克曼底盘不能原地自转，必须带前进速度。

现场处理：

```js
case "left":  return { linear_x:  0.10, angular_z:  az };
case "right": return { linear_x:  0.10, angular_z: -az };
```

同时要明确启动端口，避免 `PORT=9899` 环境变量污染 9898：

```bash
PORT=9898 nohup python3 /root/dev_ws/tools/manual_drive/app.py > /tmp/manual_drive.log 2>&1 &
```

### 3. 修复 9898/9899 启动端口混乱

问题现象：

```text
manual_drive/app.py Starting ... 0.0.0.0:9899
OSError: [Errno 98] Address already in use
```

原因：

- 在 `route_editor` 启动 Plan A 时设置过 `PORT=9899`。
- 后续在同一个 shell 中裸跑 `python3 app.py`，9898 手控继承了 `PORT=9899`。

现场处理：

```bash
unset PORT
PORT=9898 nohup python3 /root/dev_ws/tools/manual_drive/app.py > /tmp/manual_drive.log 2>&1 &

cd /root/dev_ws/tools/route_editor
export BOARD_URL=http://127.0.0.1:9898
export ROUTE_MAP_PATH=/root/dev_ws/src/origin_competition_auto/origin_competition_auto/route_map.py
PORT=9899 nohup python3 app.py > /tmp/route_editor.log 2>&1 &
```

### 4. 9898 `publisher's context is invalid`

问题现象：

```text
rclpy._rclpy_pybind11.RCLError: Failed to publish: publisher's context is invalid
```

原因：

- 手控后端里的 ROS publisher context 已失效，通常来自进程被中断、ROS context shutdown 或旧进程残留。

现场处理：

```bash
fuser -k 9898/tcp 2>/dev/null || true
fuser -k 9899/tcp 2>/dev/null || true
systemctl restart origincar-base.service
PORT=9898 nohup python3 /root/dev_ws/tools/manual_drive/app.py > /tmp/manual_drive.log 2>&1 &
```

### 5. 自动路线禁用 IMU yaw

问题现象：

- 早期日志中 `MAP_TARGET` 指令很小，但 `MAP_POSE` 往规划目标反方向漂。
- 说明 `/odom` 位置和 `/imu/data` yaw 混用后坐标系不一致。

现场处理：

```json
"use_imu_yaw": false
```

效果：

- 最新日志中起步后能连续到达 `A_DASH_01`、`A_DASH_02`、`A_DASH_03`、`A_DASH_04`。

## 最新日志关键节点

正常段：

```text
MAP_WAYPOINT_REACHED name=A_DASH_01 x=0.429 y=0.301
MAP_WAYPOINT_REACHED name=A_DASH_02 x=1.296 y=0.379
MAP_WAYPOINT_REACHED name=A_DASH_03 x=1.619 y=0.420
MAP_WAYPOINT_REACHED name=A_DASH_04 x=2.208 y=0.471
```

失控前：

```text
MAP_TARGET waypoint=A_DASH_05 dist=0.641 heading_err=0.207 linear=0.040 angular=0.207
```

失控后：

```text
MAP_POSE x=2.627 y=1.100 yaw=108.9
MAP_TARGET waypoint=A_DASH_05 dist=0.475 heading_err=-2.943 linear=0.030 angular=-0.250
```

判断：

- 车在 `A_DASH_04 -> A_DASH_05` 发生过度左转/位姿跳变。
- 之后目标点在车后侧，控制器进入大角度追点，持续饱和转向，导致现场看起来转圈。

## 下一次复跑前建议参数

先保守降低 map2d 转向，避免 A 段中后段过冲：

```bash
export CFG=/root/dev_ws/src/origin_competition_auto/config/mission_defaults.json
cp -a "$CFG" "$CFG.bak_map_follow_$(date +%Y%m%d_%H%M%S)"

python3 - "$CFG" <<'PY'
import json, sys
p = sys.argv[1]
d = json.load(open(p, "r", encoding="utf-8"))
d["use_imu_yaw"] = False
d["map_start_heading_guard_enabled"] = False
d["map_start_heading_guard_rad"] = 9.9
d["map_heading_kp"] = 0.45
d["map_max_angular"] = 0.12
d["map_waypoint_tol"] = 0.12
d["map_max_linear"] = 0.035
json.dump(d, open(p, "w", encoding="utf-8"), ensure_ascii=False, indent=2)
print("patched map follow params")
PY
```

复跑前：

```bash
curl -s -X POST http://127.0.0.1:9899/live/zero; echo
ros2 run origin_competition_auto auto_mission --config "$CFG" --start-state DRIVE_TO_QR --save-debug-images
```

观察：

- 如果 `A_DASH_04 -> A_DASH_05` 不再冲到 `y≈1.1`，继续跑。
- 如果变得转向不够，优先把 `map_heading_kp` 从 `0.45` 提到 `0.60`，不要先恢复 `map_max_angular=0.25`。
- 如果仍在同一位置失控，检查真实车体是否撞到地面线/障碍或轮子打滑，同时保存 `/tmp/manual_drive.log`、`/tmp/route_editor.log` 和完整 `auto_mission` 日志。
