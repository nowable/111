## 0. 环境（每开一个新终端,先粘这 4 行）

## 1. 开测前自检（必做,顺序别错）
```bash
systemctl restart origincar-base origincar-camera   # is-active 会骗人,必须先重启
sleep 5
ros2 node list | grep origincar_base                # 要打印 /origincar_base
ros2 topic info /cmd_vel                             # Subscription count: 要 >= 1
timeout 5 ros2 topic hz /image                       # ~30Hz(5s 后自动停)
echo "KEY_LEN=${#DASHSCOPE_API_KEY}"                 # 要 = 35
```
- 若 `/cmd_vel` 订阅=0 或没有 `/origincar_base` → 再次 `systemctl restart origincar-base`,等 5s 重验本节。

## 2. 不动车先验状态流（安全,车不动）
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --dry-run
```
看到 `STATE: IDLE → … → DONE` 即逻辑通。

## 3. 分段实车测
**规则**:每段先加 `--no-motion` 看日志确认逻辑,再去掉 `--no-motion` 实跑。车落地、前方留空、手悬 Ctrl-C(退出会自动发零速停)。

### 3.1 大厅（黑线循迹 + 躲蓝桶 + 扫码）
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state DRIVE_TO_QR
```
看:`hall black-dashed-line following enabled` → 车沿黑虚线走、遇蓝桶绕开、到码前打印 `QR_DETECTED:` + `DIRECTION: left/right`。

### 3.2 通道（黄道循迹）
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state CORRIDOR_FOLLOW --force-direction left
```
看:沿黄色居中、不蹭墙、`CORRIDOR_DONE`。

### 3.3 C 环（左/右各单独测一次）
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state LOOP_LEFT
ros2 run origin_competition_auto auto_mission --config $CFG --start-state LOOP_RIGHT
```
看:贴黄环走满一圈、**不碰绿**、`LOOP_DONE reason=yaw_complete`;路过上侧牌打印 `RING_MARKER_CAPTURED`。

### 3.4 读牌(环里自动进 CALL_LLM_API)
看:`LLM_RESULT: <中文描述>`。跑完 3.3 会自动 CAPTURE→CALL_LLM。

### 3.5 返航
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state RETURN_HOME
```
看:回到 P 区停(当前 odom 轨迹回放兜底,重点观察是否回得去)。

## 4. 全程（发车 → 回 P）
```bash
ros2 run origin_competition_auto auto_mission --config $CFG
```
看:`IDLE → … → DONE`,计时 < 175s,完赛四项(扫码 / 绕环 / 读牌 / 回 P)都过。
- 想跳过真扫码、只测后半程:加 `--mock-qr-content "ClockWise"`(=右环)或 `--mock-qr-content "AntiClockWise"`(=左环)。
- 想存调试图复盘:加 `--save-debug-images`(存到 `/root/dev_ws/debug/competition`)。

## 5. 调参（改一个键 → 重跑;改完先验 JSON）
**通用改键命令**(把 KEY/VALUE 换掉;字符串值用引号,布尔写 True/False):
```bash
python3 -c "import json;p='$CFG';d=json.load(open(p));d['lane_base_linear']=0.2;json.dump(d,open(p,'w'),ensure_ascii=False,indent=2);print('set',d['lane_base_linear'])"
python3 -c "import json;json.load(open('$CFG'));print('JSON_OK')"
```
常用旋钮(当前值 → 怎么调):

| 键 | 当前 | 怎么调 |
|---|---|---|
| `lane_base_linear` | 0.3 | 全程巡航速度;冲出就降 0.2/0.15 |
| `max_linear` | 0.5 | 速度硬上限(已够);别低于 lane_base_linear |
| `ring_bias` | 0.25 | 环贴边;蹭内侧绿→减,压外侧绿→加 |
| `bridge_timeout` | 28 | 大厅扫码后转向通道的时限;转不过来→加大 |
| `bridge_search_dir` | "left" | 找通道口的初始扫向;反了→改 "right" |
| `loop_complete_fraction` | 0.92 | 环没走满→加(如 0.97),过头→减 |
| `loop_min_time_s` | 8 | 环最短时长(防 yaw 瞬跳早停) |
| `ring_avoid_linear_scale` | 0.5 | 环上见桶减速比例 |
| `obstacle_danger_area_ratio` | 0.035 | 避障灵敏;太敏→加,太迟钝→减 |
| `hall_line_follow_enabled` | True | 关掉大厅黑线、改纯奔QR:设 False |

- **HSV(黄/蓝桶/绿/黑线)** 现场光不对导致认不出:`ros2 run origin_competition_auto vision_debug --input-file <存的帧.jpg> --config $CFG`,我远程帮你重标 `color_profiles`。

## 7. 现场排障

| 现象 | 处理 |
|---|---|
| 车不动 / `no subscriber on /cmd_vel` | `systemctl restart origincar-base`,等 5s,重验 §1 |
| `no fresh image on /image` | `systemctl restart origincar-camera`,等 4s,`timeout 5 ros2 topic hz /image` |
| 读牌无结果 / 报 key | `echo ${#DASHSCOPE_API_KEY}` 要=35;`curl -m5 https://dashscope.aliyuncs.com`(通=有网) |
| 扫不到码 | 降速、让车更正对码牌;或先 `--mock-qr-content "ClockWise"` 跳过测后段 |
| 改了 .py 没生效 | Ctrl-C 停掉重跑(纯 .py 改动不用 colcon build) |
| 改了 config 没生效 | 重跑命令(config 启动时才读) |
| 黑线/黄道乱跟 | 降 `lane_base_linear`;HSV 重标(§5 末) |
