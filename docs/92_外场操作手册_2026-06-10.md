# OriginCar 外场操作手册 2026-06-10

这份文档给今天外场测试使用。按顺序做，不要跳步。每一步只验证一个能力，失败就停在当前步骤排查。

## 0. 今天目标

今天不是直接追求完整比赛成功，而是把全过程路线拆开验证：

1. 车能从 P 点沿 A 区黑色虚线路线接近二维码。
2. 扫到二维码后，车先用 shunt 原地近似转向，把车头对齐到地图水平向左。
3. 对齐后进入 B 区通道，再进入 C 区按二维码方向绕圈。
4. 绕圈时只识别当前方向对应的一个图文牌。
5. 最后按地图返航路线回 P，失败时保留轨迹返航兜底。

当前主流程路线：

```text
P_START
  -> A_TO_QR 黑虚线到二维码
  -> SCAN_QR 扫码
  -> QR_ALIGN_HEADING 车头对齐到水平向左(180deg)
  -> QR_TO_CORRIDOR 进入 B 区
  -> B_CORRIDOR
  -> C_LOOP_LEFT 或 C_LOOP_RIGHT
  -> CAPTURE_TARGET_IMAGE
  -> CALL_LLM_API
  -> RETURN_TO_P
```

## 1. 安全规则

1. 车要动时，人手必须在电源/急停旁。
2. 每次测试前确认场地前后 1 米没人、没东西。
3. Ctrl+C 后如果车还动，马上发急停：

```bash
timeout 3 ros2 topic pub -r 20 /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
```

4. 所有地面测试先低速、短距离；不要一开始跑全流程。
5. 每次只改一个参数，改完记录现象。

## 3. 到场后第一轮检查

### 3.1 服务检查

```bash
systemctl is-active origincar-base origincar-camera cam-service
```

预期三行都是：

```text
active
```

如果底盘服务异常：

```bash
systemctl restart origincar-base.service
```

如果相机服务异常：

```bash
systemctl restart origincar-camera.service cam-service.service
```

### 3.2 话题检查

```bash
ros2 topic hz /image
ros2 topic hz /odom
ros2 topic hz /imu/data
ros2 topic echo /PowerVoltage --once
```

预期：
- `/image` 有持续频率。
- `/odom` 有持续频率。
- `/imu/data` 有持续频率。
- 电压能打印一次数值。

### 3.3 系统自检

```bash
ros2 run origin_competition_auto system_check
```

看到 `FAIL` 就先不要动车，把失败项拍照或复制发回来。

## 4. 先验证 shunt 原地近似转向

这个步骤非常重要，因为今天路线依赖“二维码后原地转到水平向左”。

### 4.1 架空测试

把车架起来，轮子离地：

```bash
ros2 run origin_competition_auto square_drive_test --side 0.1 --linear 0.03 --turn-mode shunt --stop-on-timeout
```

看现象：
- 直行段：后轮应该转。
- shunt 段：应该前进一小下、停、后退一小下、停，反复。
- 前轮方向会跟随转向命令变化。

如果 Ctrl+C 后轮子还动，立刻执行急停命令。

### 4.2 地面短距离测试

地面清空后跑：

```bash
ros2 run origin_competition_auto square_drive_test --side 0.3 --linear 0.035 --turn-mode shunt --stop-on-timeout
```

重点看日志：

```text
SQUARE_TURN_DONE index=1 mode=shunt reason=...
```

判断：
- `reason=yaw`：转角闭环成功。
- `reason=timeout` 且 `turned_deg` 很小：车转不动，先加大 shunt 速度。
- `turned_deg` 方向反了：不要继续跑，记录日志。

如果转得慢但方向正确，试：

```bash
ros2 run origin_competition_auto square_drive_test --side 0.3 --linear 0.035 --turn-mode shunt --shunt-forward-linear 0.07 --shunt-reverse-linear 0.07 --angular 0.45 --shunt-pulse-s 0.9 --stop-on-timeout
```

不要超过：
- `--shunt-forward-linear 0.08`
- `--shunt-reverse-linear 0.08`
- `--angular 0.5`

## 5. 验证二维码后水平向左对齐

先不跑真实二维码，使用强制方向做流程 dry-run：

```bash
ros2 run origin_competition_auto auto_mission --config $CFG --dry-run --start-state DRIVE_TO_QR --force-direction right
```

预期在 `DECIDE_DIRECTION` 后看到：

```text
ACTION: QR_ALIGN_HEADING target=180.0deg mode=shunt
MAP_ROUTE_START name=QR_TO_CORRIDOR
```

这说明代码流程是：

```text
扫码/定方向 -> shunt 对齐水平向左 -> 进入 QR_TO_CORRIDOR
```

如果没有这行，说明板子不是最新代码，先不要动车。

## 6. 颜色采集和 HSV 调整

今天外场灯光和宿舍不同，必须重新确认颜色。

### 6.1 采集图片

车放到赛场，分别拍：
- 黄线正样本。
- 黄线负样本。
- 深蓝障碍桶/障碍物正样本。
- 普通地面负样本。
- 橙色图文牌正样本。
- 绿色禁区正样本。

命令：

```bash
mkdir -p /root/dev_ws/datasets/origin_competition/images/train
ros2 run origin_competition_auto dataset_capture --output-dir /root/dev_ws/datasets/origin_competition/images/train --label field_sample
```

每换一个视角拍一次。拍完把文件名记录下来。

### 6.2 快速看颜色 mask

黄线：

```bash
ros2 run origin_competition_auto lane_debug --config $CFG --color yellow --save-debug-images
```

深蓝障碍：

```bash
ros2 run origin_competition_auto vision_debug --config $CFG --save-debug-images
```

如果画面里目标颜色很多但识别不到，说明 HSV 要调。先把图片抓回本地，我再帮你算 HSV。

## 7. 分段跑路线

### 7.1 A 区到二维码附近

先只测到二维码：

```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state DRIVE_TO_QR --force-direction right --save-debug-images
```

观察：
- 车是否沿 A 区黑虚线附近走。
- 是否避开深蓝障碍。
- 接近二维码后是否停下扫码。
- `DECIDE_DIRECTION` 后是否打印 `QR_ALIGN_HEADING_START`。
- 对齐后是否车头朝水平向左。

如果车走偏：
- 不要马上改 HSV。
- 先看 `MAP_POSE`、`MAP_TARGET`、`heading_err` 日志。
- 如果里程/朝向明显漂，记录起点摆放角度和偏差。

### 7.2 只测二维码后进入 B 区

如果 A 到 QR 已经稳定，再测：

```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state DECIDE_DIRECTION --force-direction right --save-debug-images
```

注意：这个命令假设车已经在二维码位置附近。它会从判定方向后开始。

预期：

```text
QR_ALIGN_HEADING_START ...
QR_ALIGN_HEADING_DONE reason=yaw
MAP_ROUTE_START name=QR_TO_CORRIDOR
```

车应先对齐水平向左，再进入 B 区入口。

### 7.3 测 B 区通道

车放到 B 区入口，车头对准通道方向：

```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state CORRIDOR_FOLLOW --force-direction right --save-debug-images
```

预期：
- 沿 B 区黄色通道走。
- 不撞墙。
- 到 C 区入口后进入 `LOOP_RIGHT`。

### 7.4 测 C 区右环/左环

右环：

```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state LOOP_RIGHT --force-direction right --save-debug-images
```

左环：

```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state LOOP_LEFT --force-direction left --save-debug-images
```

看：
- 是否压到绿色诊疗室区域。
- 是否靠外侧黄色通道走。
- 是否经过对应蓝圈位置时拍到橙色图文牌。

### 7.5 测返航

如果车在 C 区入口附近：

```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state RETURN_HOME --force-direction right --save-debug-images
```

预期先走 `RETURN_TO_P` 地图返航；地图失败时才用 trajectory fallback。

## 9. 常见问题处理

### 9.1 Ctrl+C 后车还在动

马上执行：

```bash
timeout 3 ros2 topic pub -r 20 /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
```

然后查有没有残留进程：

```bash
ps -ef | grep -E "auto_mission|square_drive_test|ros2 run" | grep -v grep
```

### 9.2 shunt 转向太慢

先试：

```bash
--shunt-forward-linear 0.07 --shunt-reverse-linear 0.07 --angular 0.45 --shunt-pulse-s 0.9
```

如果还慢，不要继续盲目加速，发 `SQUARE_TURN_DONE` 日志回来。

### 9.3 shunt 转向方向反了

立即停。发这几行日志：

```text
SQUARE_TURN_START ...
SQUARE_TURN_DONE ...
SQUARE_DONE ...
```

并说明车实际是左转还是右转。

### 9.4 二维码后没有水平向左

看日志：

```text
QR_ALIGN_HEADING_START target_deg=180.0 current_deg=... delta_deg=...
QR_ALIGN_HEADING_DONE reason=...
```

如果没有这两行，代码/配置不是最新。
如果 `reason=timeout`，说明 shunt 没转够，先单独调 shunt。
如果 `current_deg` 明显不对，说明 map 起点姿态或 IMU yaw 有问题。

### 9.5 颜色识别不稳定

先采样，不要现场乱改多个 HSV：

```bash
ros2 run origin_competition_auto dataset_capture --output-dir /root/dev_ws/datasets/origin_competition/images/train --label hsv_debug
```

把图片抓回来后再算 HSV。现场只记录“黄线识别不到/深蓝误识别/橙牌识别不到”等现象。
