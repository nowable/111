# 比赛规则文档 — 医院赛道（OriginCar / RDK X5）

> 唯一事实来源：赛道规则与任务要求，2026-06-13/14 现场逐条确认整理。
> 实现（寻迹 / 避障 / 状态机）一律以本文档为准。

## 0. 一句话总览

出 P → 扫码（定 C 环顺/逆）→ 上通道 → C 环绕满一圈（途中拍一块图文牌交 LLM 描述）→ 回 P 停车。
**纯导航 + 感知任务，无取放、无机械臂、无任何执行动作。**

## 1. 场地与区域

| 区域 | 颜色 / 尺寸 | 规则 |
|---|---|---|
| **A 大厅** | 浅蓝，5×2 m | 含发车/停车点 P、二维码立牌、非固定障碍；车可自由行驶但须躲障 |
| **通道** | 黄，车道宽 1 m | 仅可在黄色通道内行驶；两侧黄色网格 = 墙，禁碰 |
| **C 诊疗室区** | 浅绿，5×2.5 m | 内含 0.5 m 宽黄色环道，**车仅可走黄环**；浅绿（诊疗室）禁入禁碰 |

- **P 点**：深蓝方块 + 一圈白色边框 + 中央 “P” 字（起点 & 终点停车识别）。
- **场地边界**：外侧一层白色围栏。

## 2. 场上物体（实物以此为准，地图图例仅示意）

| 物体 | 实物 | 检测 |
|---|---|---|
| 障碍物 | **深蓝标志桶**，位置非固定，黄区也可能有；**绝不堵死、必留通路、不逼车上绿** | **YOLO** |
| 二维码（= 任务发布点） | **白底黑码立牌**（同一处） | QR 解码（pyzbar） |
| 图文标记牌 | **左上 / 右上角各一块，内容相同**；**是一张图片、需描述** | **qwen-vl（云端，现场连热点有网）** |
| 物品放置区 A/B/C | **本赛事不使用** | — |
| 黑色虚线 | 大厅地面引导线 | 仅作“无目标时兜底大方向” |

## 3. 任务全流程（状态序列）

1. **起步 @P**：车头大概率已能看见二维码立牌。
2. **奔二维码**：朝白底黑码立牌行驶，YOLO 躲蓝桶。
3. **扫码**：解码 → 内容决定 **C 环顺时针 / 逆时针**（对应见 §6）。
4. **奔通道口**：朝大片黄色通道口行驶，躲桶。
5. **上通道**：黄色车道居中（宽 1 m），勿碰两侧黄网格墙。
6. **进 C 环**：通道从环正下方接入，按 §3 方向转入。
7. **绕环一圈**：沿 0.5 m 黄环行满一整圈，**全程不碰绿 / 墙**。
   - 沿上侧直行时正对角上的牌 → **拍照交 qwen-vl 描述**；两块相同，读一块即可，可靠近取清晰。
8. **下通道**：从同一口回到通道、下行。
9. **回 P**：回大厅，识别 P 标志 → 停车 → **完赛**。

## 4. 完赛判定（硬指标，四项缺一不算完赛）

- [ ] 扫到二维码
- [ ] C 环绕满一整圈、方向正确
- [ ] 读出 ≥1 块图文牌（LLM 给出图片描述）
- [ ] 回到 P 点停车

## 5. 计分与罚时

- **第一优先 = 完赛**；完赛者再按**用时排名**（越快越好）。
- **罚时**：车体进入**绿色（诊疗室）/ 中间禁区**或**撞墙**，在完赛用时上**追加罚时**。
- **限时 180 s / 轮**；全程巡航目标 ~**0.3 m/s**。

## 6. 二维码语义（已查代码确定，无需再翻实测）

解析器 `decision_parser.py`（`auto_mission` 与 `qr_parse_debug` 共用）把码内容统一解析成 `left` / `right`：

| 解析结果 | 同义词（都认） | → C 环走法 |
|---|---|---|
| **`right`** | 顺时针 / cw / clockwise / 右 / 右转 / 右圈 / R | **右环**（C_LOOP_RIGHT / 顺时针） |
| **`left`** | 逆时针 / ccw / 左 / 左转 / 左圈 / L | **左环**（C_LOOP_LEFT / 逆时针） |

**码内容支持多种写法（任选，推荐 JSON）：**
- JSON：`{"direction":"left"}`、`{"cmd":"right"}`、`{"route":"L"}` / `{"route":"R"}`
- 键值 / URL：`direction=left`、`dir=R`、`turn=right`、`?direction=left`
- 自然语言 / 中文：`left` / `right`、`左转` / `右转`、`逆时针` / `顺时针`

**失败保护：** 解析为空、或左右同时出现冲突 → 主流程**停车不盲走**（`decision_parser` 的 ambiguous 分支）。

**现场实测真码（已解码 + 解析验证 2026-06-14）：** 两张牌分别印
- `ClockWise` → 解析 `right` → **右环 / 顺时针**
- `AntiClockWise` → 解析 `left` → **左环 / 逆时针**

主流程 `SCAN_QR`/`_confirm_direction` **无置信度门槛**（只看 direction∈{left,right} + 连续 N 票一致防抖），真码 0.45 置信度照样接受，可直接用。

## 7. 关键约束（硬限制）

- **底盘 = 阿克曼，不能原地转** → 所有转向必须连续小角度，禁止依赖原地掉头。
- **C 环仅 0.5 m 宽** → 过角时阿克曼转弯半径需实车低速验证不蹭绿。
- **无可靠全局定位**（odom 真车上会瞬跳，已弃用）→ 全程靠视觉循迹 + 可见目标驱动。
- **读牌走云端 VLM 且为完赛硬指标** → 必须加超时 + 重试，并以“两块相同牌 = 两次机会”兜底。

## 8. 已确定的实现方向（本次讨论结论）

- **架构 = 一个“全局避障层”（YOLO，全程常开）叠加在“每段的转向意图”上：**
  - 大厅：**奔可见目标**（二维码 → 通道口），黑虚线兜底，躲桶；
  - 通道 / C 环：**黄色车道循迹**；
  - C 环上侧：机会式拍牌 → qwen-vl 描述；
  - 回程：循迹 / 奔 P → 识别 P 停车。
- **零新模型训练**；颜色阈值（黄道 / 黑线等）从已有 400+ 张实拍中提取。
- 状态机沿用现有 `auto_mission.py` 骨架，把各状态控制器由“map 航点”换成“循迹 / 奔目标”，避障从挂在 map 上提为独立全局层。
