## 步骤 0 ─ 连上车（小终端，3 分钟）

**① 找板子 IP**：手机热点 → 已连接设备列表，找主机名含 `rdk` 的那台，记下它的 IP（连手机热点多半是 `192.168.x.x`，不再是家里的 10.20.240.x）。下文用 `<IP>` 代替。

**② SSH 连进去**（小终端）：
```bash
ssh root@<IP>
```

**③ 进去后第一件事，整段粘贴**（每开一个新终端都要粘一次）：
```bash
source /opt/ros/humble/setup.bash && source /opt/tros/humble/setup.bash && source /root/dev_ws/install/setup.bash
CFG=/root/dev_ws/src/origin_competition_auto/config/mission_defaults.json
echo "CFG=$CFG"
```
看到 `CFG=/root/dev_ws/...` 打印出来即成功。`ros2` 命令找不到 = 这三行没 source，重粘一次。

## 步骤 4 ─ 确认 YOLO 避障活着（不动车，2 分钟）

避障主力是进程内 YOLO11n（BPU），挂了自动回退 HSV。跑一次任务看启动日志：
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --no-motion --start-state SCAN_QR
```
启动几行里**必须看到**：
```
local YOLO backend=bpu model=...
```
- 看到 → YOLO 主线正常，`Ctrl+C` 退出，继续。
- 看到 `local YOLO load failed, falling back to HSV` → bin 路径错/缺失：`ls /root/dev_ws/models/` 核对 `$CFG` 里 `yolo_local_model` 路径。
- 跑中途出现 `local YOLO inference failed, disabling` → 本次任务永久回退 HSV 不恢复，重跑即重新加载；连续两次就把 `yolo_local_conf` 调高或 `yolo_local_model` 置空回纯 HSV。

## 步骤 6 ─ arc 150° 转向计时（随步骤 8① 一起测，**不能冷启动单独测**）

⚠️ arc 对齐需要"地图位姿"，它只在 DRIVE_TO_QR 走到码板、MAP_REANCHOR 重锚定时才建立——冷启动 `DECIDE_DIRECTION` 会报 `QR align requested but map pose is unavailable`（实测确认）。所以 **arc 计时跟着步骤 8① 真实 DRIVE_TO_QR 跑**：车扫完码转向时，秒表掐这两行之间：
```
QR_ALIGN_HEADING_START ... mode=arc
QR_ALIGN_HEADING_DONE reason=arc_complete
```
- ≤8s：通过，方案第五节速度账成立。
- >10s：**arc 模式转向时间由角速度定，跟 linear 无关**（清单§2"调linear"是 shunt 残留，别信）。提速旋钮是 `--turn-angular`，加大即可，时间约反比缩短：
  ```bash
  ros2 run origin_competition_auto auto_mission --config $CFG --start-state DECIDE_DIRECTION --force-direction left --turn-angular 0.6
  ```
  别超 ~0.8（底盘转向上限）。定下值后写进 `$CFG` 的 `fixed_route_turn_angular`。
- 记录实测：______ s @ turn-angular=______

> 注：最干净的数字其实出现在步骤 8 跑真实 DRIVE_TO_QR 全程时（几何是真的）。这里只是先单独掐一下。

## 步骤 7 ─ Plan B 录 5 段基线（手机 9898，45 分钟，核心产出）

这是 Plan A 失败时的兜底。**手机开 `http://<IP>:9898`**，录制全程在网页上点。

**每段录制前**：把车物理摆到该段起点；坐标是 odom 相对帧，起点摆准最重要。

5 段（段名照填，转换脚本认这个名）：

| 顺序 | 段名 | 路线 |
|------|------|------|
| ① | `A_TO_QR` | P 点 → A 区沿边 → QR 停车点 |
| ② | `B_CORRIDOR` | QR 停车点 → B 黄色通道 → C 入口 |
| ③ | `C_LOOP_CW` 或 `C_LOOP_CCW` | C 环，**按当日扫码方向选一个**（顺=CW / 逆=CCW） |
| ④ | `RETURN_HOME` | 拍照完 → 回 P 点 |

> 清单写的是 5 段，这里 B 区合成一段录（QR→B→C入口连着走更顺）；C 环当天只录用得上的那个方向即可，省时间。

**录制操作**（网页上）：填段名 → 点「⏺ 开始」→ 方向键手控走完该段 → 点「⏹ 停止」。自动存到板上 `/root/dev_ws/tools/manual_drive/recordings/<段名>_<时间戳>.json`。

**每段录完立即转航点检查形状**（小终端，纯 python3 不需 ROS）：
```bash
cd /root/dev_ws/tools/manual_drive
python3 convert_recording.py recordings/A_TO_QR_<时间戳>.json
```
生成 `*_waypoints.json`（可灌 Plan A）+ `*_snippet.py`。形状不对就重录。

## 步骤 8 ─ 自动分段跑 + 读漂移（地面慢速，60 分钟）

**车会动了，手守急停。** 默认 0.04 m/s 很慢，给足反应时间。逐段过，过了再连。

**① A 段（最关键，读漂移）**——车放 P 点、二维码摆右侧：
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state DRIVE_TO_QR
```
盯三行日志：
```
QR_BOARD_REACHED area=... offset=...     # 到板停车
DIRECTION_LOCKED ...                      # 锁方向
MAP_REANCHOR ... drift=______             # ★ 这个 drift 数字记下来 ★
```
**drift ≤ 0.3 m = 开环精度够用，全场信心成立**（这是今天最该带回的数）。drift 很大就说明里程计漂得凶，返航得靠地图重锚定救。

**② 扫码方向**——左右两个方向各放一次二维码，确认 `DIRECTION` 认对、转向正常。

**③ B 段 + C 环**（C 环先默认 0.05）：
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state CORRIDOR_FOLLOW --force-direction right
ros2 run origin_competition_auto auto_mission --config $CFG --start-state LOOP_RIGHT
```
（当天扫码是 left 就换 `--force-direction left` / `--start-state LOOP_LEFT`。）

**④ 返航段**：
```bash
ros2 run origin_competition_auto auto_mission --config $CFG --start-state RETURN_HOME
```
默认 `map2d_fallback`：先走地图路线 `RETURN_TO_P`（经 B_MID 中点），失败才轨迹重放。确认能回 P 点。

> 哪段不稳就停在哪段反复调（调参查附录 A），别带着问题往下跑。

## 步骤 9 ─ 提速（每档记一次，30 分钟）

各段都过了再提速。`nano $CFG`，把 `map_max_linear` + `cruise_linear` 一起改（map 路线模式真正生效的是 `map_max_linear`，`cruise_linear` 一起改保险）：

- 第一档 **0.04 → 0.10**：重跑 A 段（步骤 8①），看到点准度、有没有冲过头。
- 没问题再 **0.10 → 0.15**：跑全程一次。
- 冲过头/压点 → 看是否要收 `map_waypoint_tol`（默认 0.18，**键名带 map_ 前缀**）。

每档记录：到点准度、有无冲过头。硬上限 `max_linear=0.1`——要上 0.15 得先把它也抬上去。

## 附录 A ─ 调参速查（现象 → 改哪个键）

全部在 `$CFG` 改，改完直接重跑即生效（不用重新编译）。逐键详解看 `mission_defaults.annotated.jsonc`。

| 现象 | 改哪个键 | 往哪调 |
|------|----------|--------|
| 车太慢/太快 | **`map_max_linear`(普通路线)、`map_loop_linear`(C环)** | 默认 0.04/0.05；`cruise_linear`/`lane_base_linear`/`loop_linear` 仅非-map 模式生效，**map 路线模式下改它们无效** |
| 黄线认不准 | `color_profiles.yellow` 的 HSV | 见步骤 5 |
| 循迹蛇行 | `lane_kp` 调小 / `lane_kd` 调大 | 一次动一个 |
| 循迹迟钝压线 | `lane_kp` 调大 | |
| 二维码停太远 | `qr_min_area_ratio` 调大 | 越大停越近 |
| 二维码停太近/撞 | `qr_min_area_ratio` 调小 | |
| 方向老跳变 | `qr_confirm_count` 调大 | 默认 3，要更稳设 5 |
| 绕环不到一圈停 | `loop_complete_fraction` 调大 | 占 360° 比例 |
| 绕环绕过头 | `loop_complete_fraction` 调小 | |
| 绕环蹭中间禁区 | `ring_bias` | 调贴边偏置 |
| 避障完全没反应 | `obstacle_avoid_enabled` | 板上当前 false，要避障设 true |
| 避障误检多/乱触发 | `yolo_local_conf` | 调高 0.5→0.6 减误检；仍乱把 `yolo_local_model` 设 `""` 回纯 HSV |
| 避障太敏感/迟钝(HSV兜底) | `obstacle_danger_area_ratio` | 大=迟钝 小=敏感；仅 YOLO 挂走 dark_blue 时有效 |
| 返航不准 | `return_waypoint_stride`、`return_heading_tol` | 默认 `map2d_fallback`(先地图路线，失败才轨迹重放) |
| arc 转向太慢 | `fixed_route_turn_angular` | 默认 0.4，加大缩短转向时间(别超 0.8)；见步骤 6 |
| shunt_* 系列 | — | **已废弃，转向固定 arc，别动** |
| 整局时间不够 | `max_runtime` | 当前 175，规则上限 180 |

## 附录 B ─ 排查表（出问题怎么办，完整版）

> 步骤之间的「一分钟应急速查」是这张表的常用子集，下面是完整版。

| 现象 | 原因 / 怎么办 |
|------|--------------|
| `ros2` 命令找不到 | 没 source → 重粘步骤 0③ |
| 车不动、`/cmd_vel` 没反应 | 底盘串口 USB 抖动崩 → `systemctl restart origincar-base.service` |
| 从 P 出发只走很短就停 | 若用 `competition_run --profile field_low_speed` 那是 2s 测试档；完整跑用 `auto_mission --config $CFG`；仍这样看有无 `QR_BOARD_REACHED`/`DIRECTION_LOCKED`/`QR scan timeout` |
| 预检 `no fresh image` | 相机没数据 → `systemctl restart origincar-camera.service`，再 `ros2 topic hz /image` |
| 预检 `no subscriber on /cmd_vel` | 底盘节点没起 → 重启 origincar-base |
| 预检 `no voltage` / 电压低 | 换电池或充电 |
| AI 报 403 access_denied | 确认用 `qwen-vl-plus`(不是 max) + 已 `export DASHSCOPE_API_KEY` |
| AI 报 missing_env | 没 export 密钥 或 没带 `--config` |
| 二维码扫不出 | 光照/距离 → `qr_capture` 单独抓帧；调 `qr_min_area_ratio` |
| 循迹乱打方向 | 黄色 HSV 太松，把别的当黄线 → 步骤 5 收紧 |
